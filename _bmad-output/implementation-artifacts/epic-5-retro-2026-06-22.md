## What went well

- **Consent-aware send path** — Server skips non-consented clients; segment preview matches deliverable count.
- **Review hardened real bugs** — Empty segment guard, AND filter merge, all-skipped → Failed, WhatsApp log before `window.open`.
- **Operator-safe compose** — Send blocked with reason when no emailable consented recipients; subject/body length validated.
- **Timeline continuity** — Campaign sent + WhatsApp events merge into Epic 3 timeline; reports follow-up coverage updated.
- **Campaign detail for failures** — Operators can diagnose skipped/failed recipients post-send.

## What we learned

- **Consent is data, not just send logic** — Legacy forms without consent fields blocked campaigns until Epic 6 backfill (production lesson).
- **SendGrid accept ≠ delivery** — Sandbox/processed states confused operators; DNS verification is outside app but must be documented.
- **Sequential send loop is MVP-fine** — Background jobs deferred until segment size forces the issue.
- **Destructive timeline append needs UX guard** — Epic 5 shipped open save on follow-up; Epic 6.4 added dirty-state guard after duplicate-entry incidents.
- **CI gap persists** — SendGrid validator unit tests exist; workflow wiring still missing.
